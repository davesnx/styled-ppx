# Rule: `css` extension

Generates a `Css.rule{:rescript}` `Css.rule{:reason}` given a single CSS rule as string. The `%css{:rescript}` `[%css]{:reason}` extension is useful in combination with the [Array API](./array) to compose and re-use rules.

## Example

```rescript
let rule: Css.rule = %css("display: block")
```
```reason
let rule: Css.rule = [%css "display: block"];
```

## Features

- Only single properties are allowed, e.g., `color: blue{:styled-ppx-css}`, and multiple declarations, e.g., `:hover { color: blue }{:styled-ppx-css}` or `color: blue; background-color: red{:styled-ppx-css}`, are not allowed. If you need to use a selector or use more than one rule, use `%cx{:rescript}` `[%cx]{:reason}` instead.
- The semi-colon in the last position is optional. `display: block;{:styled-ppx-css}` and `display: block{:styled-ppx-css}` are the same.
- Curly braces aren't allowed. `{ display: block }{:styled-ppx-css}` isn't valid.
