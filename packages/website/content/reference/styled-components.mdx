# Styled components

`styled.{{tag}}` is the extension that allows you to create React Components from the styles of the element.
All HTML Elements are available under `styled.{{tag}}`. An example of it will be:

```reason
module StyledComponent = [%styled.div {|
  display: flex;
  justify-content: center;
  align-items: center;
  height: 100vh;
  width: 100vw;
|}];

<StyledComponent> {React.string("Hello from the style")} </StyledComponent>;
```

```mlx
module StyledComponent = [%styled.div {|
  display: flex;
  justify-content: center;
  align-items: center;
  height: 100vh;
  width: 100vw;
|}]

let app = <StyledComponent> (React.string "Hello from the style") </StyledComponent>
```

The styles are [extracted at compile time](/how-it-works) like any other block: the component simply renders the given tag with the generated class names applied.

### React props (makeProps)

Styled components expose all the React component props within their API. For example, an input will expose all the props of the input element:

```reason
module Input = [%styled.input {|
  border: 1px solid #cccccc;
  border-radius: 4px;
  padding: 8px;

  &:focus {
    border-color: #aaaaaa;
  }
|}];

<Input
  placeholder="Enter your name..."
  maxLength=255
  onChange={_ => ()}
  onBlur={_ => ()}
/>;
```

```mlx
module Input = [%styled.input {|
  border: 1px solid #cccccc;
  border-radius: 4px;
  padding: 8px;

  &:focus {
    border-color: #aaaaaa;
  }
|}]

let app =
  <Input
    placeholder="Enter your name..."
    maxLength=255
    onChange=(fun _ -> ())
    onBlur=(fun _ -> ())
  />
```

<Callout>
  You can extend the styles from a component via the `className` prop.
</Callout>

#### Extend styles with `className`

Styled components also accept a `className` prop, appended after the generated classes. Prefer composing with [`CSS.merge`](./runtime#composition); `className` is the escape hatch for classes coming from outside styled-ppx (third-party CSS, integration code).

```reason {6}
<Input
  placeholder="Enter your name..."
  maxLength=255
  onChange={_ => ()}
  onBlur={_ => ()}
  className="extra-classname"
/>
```

```mlx {6}
let app =
  <Input
    placeholder="Enter your name..."
    maxLength=255
    onChange=(fun _ -> ())
    onBlur=(fun _ -> ())
    className="extra-classname"
  />
```

The resultant HTML contains the generated atomic class names (one per declaration) plus the className passed by props:

```html
<input class="css-1qbcma3-Input css-tokvmb-Input css-8i02mp-Input css-1ru12dh-Input extra-classname" maxlength="255" placeholder="Enter your name...">
```

### Derive styles from props

`styled.{{tag}}` can also be defined by a function returning the CSS string. The function's labelled arguments become props on the component, on top of all the props from the element:

```reason
module Dynamic = [%styled.div (~color, ~background) => {|
  color: $(color);
  background-color: $(background);
|}];

<Dynamic color={CSS.hex("#EB5757")} background={CSS.hex("#516CF0")}>
  {React.string("Hello!")}
</Dynamic>;
```

```mlx
module Dynamic = [%styled.div fun ~color ~background -> {|
  color: $(color);
  background-color: $(background);
|}]

let app =
  <Dynamic color=(CSS.hex "#EB5757") background=(CSS.hex "#516CF0")>
    (React.string "Hello!")
  </Dynamic>
```

All the rules from [Interpolation](./interpolation) apply here. In the example, `color` and `background` are `CSS.Types.Color.t{:reason}` since it gets inferred from the CSS property. It's called type holes: https://twitter.com/davesnx/status/1552305210558742528

The styles (including the interpolated ones) are [extracted at compile time](/how-it-works#dynamic-values-css-custom-properties) like any other block. Each interpolated declaration references a CSS custom property, and the component supplies the values through its inline `style` attribute.

A few constraints:

- The returned expression (the last expression of the function) needs to be a string literal: it can't be a reference or a function call. The CSS must be visible to the ppx at compile time, otherwise it fails with `Extracted dynamic styled components require a CSS string body.`
- A function payload with no labelled arguments is rejected (`A dynamic component without props doesn't make much sense. This component should be static.`). Use the plain string form instead.

#### Name collision

If a labelled argument has the same name as one of the element's props (`makeProps`), styled-ppx overrides it and the prop is replaced entirely.

For example, a component defined with a `~size` argument overrides `size` from the element's props.
