# Melange

Ensure that you have set up a Melange project beforehand. If you haven't done so yet, refer to the Melange getting started [documentation](https://melange.re/v3.0.0/getting-started.html).

This guide assumes that you'll use opam and dune.

### Install

```bash
opam install styled-ppx
```

### Packages available

- `styled-ppx` is the ppx that transforms `[%css ""]`, `[%styled.div ""]`, `[%styled.global ""]` and `[%keyframe ""]`
- `styled-ppx.melange` is the runtime library
- `styled-ppx.generate` is the CLI that writes the extracted stylesheet

### Usage

1) Add `styled-ppx` under dune's preprocess pps
2) Add `styled-ppx.melange` under libraries

in a dune library with `(modes melange)`:

```diff
(library
  (name ...)
  (modes melange)
  (libraries
+   styled-ppx.melange
    reason-react)
  (preprocess
    (pps
+     styled-ppx
      melange.ppx
      reason-react-ppx)))
```

or in the `melange.emit` stanza:

```diff
(melange.emit
  (libraries
+   styled-ppx.melange
    reason-react)
  (preprocess
    (pps
+     styled-ppx
      melange.ppx
      reason-react-ppx)))
```

Note: `reason-react` and `reason-react-ppx` are optional, and only needed if you use styled components (`[%styled.div {||}]`).

### Generate the stylesheet

styled-ppx extracts all CSS at compile time — nothing injects styles at runtime. You need one dune rule that collects the extracted CSS into a `styles.css` file:

```dune
(rule
 (mode promote)
 (target styles.css)
 (deps
  (glob_files_rec *.pp.ml))
 (action
  (run styled-ppx.generate --output %{target} --log info %{deps})))
```

Then link it from your HTML like any other stylesheet:

```html
<link rel="stylesheet" href="/styles.css" />
```

<Callout type="warning">
  Without this rule your app compiles fine but renders unstyled: the class
  names exist on the elements, but no stylesheet defines them. See
  [Setting up the dune rules](/getting-started/dune-setup) for the full guide,
  including where to place the rule and how to serve the file.
</Callout>

### Example

```reason
/* This is a ReasonReact component with those styles applied */
module Link = [%styled.a (~color=CSS.hex("4299E1")) => {|
  font-size: 1.875rem;
  line-height: 1.5;
  text-decoration: none;
  margin: 0px;
  padding: 10px 0px;
  color: $(color);
|}];

/* This is a CSS.styles value carrying the generated classNames */
let layout = [%css {|
  display: flex;
  width: 100%;
  height: 100%;
  justify-content: center;
  align-items: center
|}];

/* Later in a component: pass it via the `styles` attribute */
<div styles=layout>
  <Link
    color={CSS.hex("333333")}
    href="https://sancho.dev"
    rel="noopener noreferrer">
    {React.string("sancho.dev")}
  </Link>
</div>;
```

The `styles` attribute is rewritten by the ppx into `className` (and `style`, when the block interpolates values). If you need the raw strings — for example outside JSX — use `CSS.className(layout)` and `CSS.styles(layout)`.
