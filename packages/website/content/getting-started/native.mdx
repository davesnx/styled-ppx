# OCaml (native)

This guide assumes that you'll use [opam](https://github.com/ocaml/opam) and [dune](https://github.com/ocaml/dune) and have a server or native process running, rendering HTML with [server-reason-react](https://github.com/ml-in-barcelona/server-reason-react).

### Install

```bash
opam install styled-ppx
```

### Packages available

- `styled-ppx` is the ppx that transforms `[%css ""]`, `[%styled.div ""]`, `[%styled.global ""]` and `[%keyframe ""]`
- `styled-ppx.native` is the runtime library for server-reason-react
- `styled-ppx.generate` is the CLI that writes the extracted stylesheet

### Usage

Add `styled-ppx` with the `--native` flag under dune's preprocess pps for any library or executable, and add `styled-ppx.native` as a library:

```diff
(library
  (name ...)
  (libraries
+  styled-ppx.native
   server-reason-react.react
   server-reason-react.reactDom)
  (preprocess
   (pps
+   styled-ppx
+   --native
    server-reason-react.ppx
    server-reason-react.melange_ppx)))
```

The `--native` flag tells the ppx to generate code targeting server-reason-react instead of Melange.

### Generate the stylesheet

All CSS is extracted at compile time. Add a dune rule that collects it into a `styles.css`:

```dune
(rule
 (mode promote)
 (target styles.css)
 (deps
  (glob_files_rec *.pp.ml))
 (action
  (run styled-ppx.generate --output %{target} --log info %{deps})))
```

Serve the file from your server like any static asset and link it in the document you render:

```reason
<head>
  <link rel="stylesheet" href="/styles.css" />
</head>
```

<Callout type="info">
  Because styles are extracted at build time, there is **no runtime
  stylesheet, registry, or render-order concern**: you don't need to render
  your app before collecting styles, and there's nothing to flush or hydrate.
  Link the file and render. See
  [Setting up the dune rules](/getting-started/dune-setup) for the full guide.
</Callout>

### Example

```reason
/* This is a server-reason-react component with those styles applied */
module Link = [%styled.a (~color=CSS.hex("4299E1")) => {|
  font-size: 1.875rem;
  line-height: 1.5;
  text-decoration: none;
  margin: 0px;
  padding: 10px 0px;
  color: $(color);
|}];

/* This is a CSS.styles value carrying the generated classNames */
let layout = [%css {|
  display: flex;
  width: 100%;
  height: 100%;
  justify-content: center;
  align-items: center
|}];

/* Later in a component: pass it via the `styles` attribute */
let app =
  <div styles=layout>
    <Link
      color={CSS.hex("333333")}
      href="https://sancho.dev"
      rel="noopener noreferrer">
      {React.string("sancho.dev")}
    </Link>
  </div>;
```

The `styles` attribute is rewritten by the ppx into `className` (and `style`, when the block interpolates values). If you need the raw strings, use `CSS.className(layout)` and `CSS.styles(layout)`.

### Global styles

`[%styled.global]` defines document-level rules (resets, `@font-face`, `:root` variables). The generated module is a React component — render it once in your document:

```reason
module Reset = [%styled.global {|
  * {
    margin: 0;
    box-sizing: border-box;
  }
|}];

module Document = {
  [@react.component]
  let make = (~children) => {
    <html>
      <head>
        <link rel="stylesheet" href="/styles.css" />
      </head>
      <body>
        <Reset />
        children
      </body>
    </html>;
  };
};

/* Let's assume we are using dream (from https://github.com/aantron/dream) */
let some_server_side_handler = _request => {
  Dream.html(ReactDOM.renderToString(<Document> <App /> </Document>));
};
```

The static rules themselves live in the extracted `styles.css`; the rendered `<Reset />` element only supplies `:root` custom-property values when the global block interpolates runtime values (and renders an empty `<style>` otherwise).

### Sharing styles with a Melange client

If your UI code compiles for both native (SSR) and Melange (client), preprocess each build with the matching runtime (`styled-ppx.native` + `--native` vs `styled-ppx.melange`). The extracted CSS is identical for both targets, so a single `styled-ppx.generate` rule and a single `styles.css` serve both.
