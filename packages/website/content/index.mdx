### Type-safe styled components for OCaml and Reason. Compile-time CSS, minimal runtime.

**styled-ppx** is a [ppx](https://tarides.com/blog/2019-05-09-an-introduction-to-ocaml-ppx-ecosystem) and a library that brings styled components to OCaml and Reason, for both [Melange](https://melange.re) and native ([server-reason-react](https://github.com/ml-in-barcelona/server-reason-react)). Write plain CSS inside your components and get **React components** or **class names** back: parsed, validated and type-checked at compile time.

```reason
module Center = [%styled.div {|
  height: 100vh;
  width: 100vw;
  display: flex;
  align-items: center;
  justify-content: center;
|}];

<Center> {React.string("Hello from the future!")} </Center>
```

```mlx
module Center = [%styled.div {|
  height: 100vh;
  width: 100vw;
  display: flex;
  align-items: center;
  justify-content: center;
|}]

let app = <Center> (React.string "Hello from the future!") </Center>
```

<Ml>The OCaml examples use [mlx](https://github.com/ocaml-mlx/mlx), the OCaml syntax dialect with JSX support.</Ml>

## Catch CSS errors at compile time

styled-ppx ships its own CSS parser and type-checker. Every property and value is validated against the CSS spec while you compile: a typo in a property name, an invalid value, or a wrongly-typed interpolation is a **compile error**, not a broken page.

Dynamic values are welcome: any OCaml or Reason expression goes inside `$( ... )`, [type-checked](/reference/interpolation) against the property it's used in.

## Ship a static stylesheet

All CSS is extracted while your project compiles:

1. The ppx splits your CSS into **atomic rules** (one class per declaration) and hashes each one into a content-addressed class name, so repeated declarations across your app dedupe into one rule.
2. A build step ([`styled-ppx.generate`](/how-it-works#the-aggregator-styled-ppxgenerate)) collects every rule in your project and writes a single, deduplicated `styles.css`.
3. At runtime, your components only carry class names. Interpolated values become CSS custom properties supplied through the `style` attribute.

The runtime is minimal: it only carries class names and supplies variable values.

Class names are human-readable in development (`css-<hash>-<binding>`) and minified in production. Read more in [How it works](/how-it-works).

## Compose styles as code

Styles are ordinary values of the language: pattern match over them, merge them, pass them as labelled arguments. There is no styling DSL to learn, and your bundle carries the minimum runtime.
